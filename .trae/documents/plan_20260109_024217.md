## 问题分析
从日志中可以看到，无法写入Neo4j数据库的原因是尝试写入已经存在的实体，违反了唯一性约束：
```
Node(12) already exists with label `Entity` and property `name` = 'Supplier Clean Water Program'
```

**根本原因**：在`graph_builder.py`中，每次写入数据库时，程序都会将`all_entities`中的所有实体写入，而不仅仅是新增的实体。

## 解决方案
修改`graph_builder.py`文件，确保每次只写入新增的实体，而不是所有实体。

### 实现步骤

1. **添加新增实体追踪机制**：
   - 在`build_graph`方法中，添加一个`new_entities`字典或列表
   - 用于存储每个批次中新增的实体

2. **修改实体处理逻辑**：
   - 在处理实体时，记录哪些实体是新增的
   - 将新增实体添加到`new_entities`字典中

3. **修改写入数据库逻辑**：
   - 每次写入数据库时，只写入`new_entities`中的实体
   - 写入完成后，清空`new_entities`字典

4. **保留现有去重机制**：
   - 继续使用`all_entities`字典进行去重
   - 确保整个处理过程中实体的唯一性

### 关键代码修改

1. **在`build_graph`方法中初始化新增实体字典**：
   ```python
   # 存储所有实体（去重）
   all_entities = {}
   # 存储当前批次新增的实体
   new_entities = {}
   # 存储临时关系
   temp_relationships = []
   ```

2. **修改实体处理逻辑，记录新增实体**：
   ```python
   entity_key = (entity_name, entity_type)  # 使用清理后的名称和类型作为唯一键
   if entity_key not in all_entities:
       # 创建新的Entity对象并添加到字典中
       entity_obj = Entity(
           name=entity_name,
           type=entity_type,
           properties={"description": entity_desc}
       )
       all_entities[entity_key] = entity_obj
       new_entities[entity_key] = entity_obj  # 记录为新增实体
   ```

3. **修改写入数据库逻辑，只写入新增实体**：
   ```python
   # 将当前批次的实体存储到Neo4j
   logging.info(f"批次 {current_batch}: 正在将 {len(new_entities)} 个新增实体存储到Neo4j...")
   entity_list = list(new_entities.values())
   entities_created = self._batch_create_with_retry(
       entity_list,
       self.neo4j_manager.batch_create_entities
   )
   total_entities_created += entities_created
   ```

4. **写入完成后清空新增实体字典**：
   ```python
   # 清空临时关系列表和新增实体字典
   temp_relationships = []
   new_entities = {}
   ```

## 预期结果
- 每次写入数据库时，只会写入新增的实体
- 不会尝试写入已经存在的实体
- 避免违反唯一性约束
- 提高写入数据库的效率
