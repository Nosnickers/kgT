## 知识图谱文本化与嵌入向量化实施方案

### 技术栈
- **嵌入模型**：sentence-transformers（all-MiniLM-L6-v2）
- **向量数据库**：ChromaDB
- **生成模型**：Ollama（deepseek-r1:8b）
- **框架**：LangChain

### 实施步骤

#### 阶段1：创建文本生成模块（src/text_generator.py）
- 从 Neo4j 读取所有实体和关系
- 从 CSV 文件读取 chunk_id 对应的原始文本
- 为每个实体生成文本描述和上下文文本
- 为每个关系生成文本描述
- 保存为 JSON 文件

#### 阶段2：创建嵌入管理模块（src/embedding_manager.py）
- 使用 sentence-transformers 加载 all-MiniLM-L6-v2 模型
- 实现批量文本嵌入
- 实现嵌入缓存机制
- 支持自定义嵌入模型

#### 阶段3：创建向量存储模块（src/vector_store.py）
- 使用 ChromaDB 作为向量数据库
- 存储实体向量（基于 text_description）
- 存储关系向量（基于 relationship_text）
- 存储元数据（实体类型、关系类型、chunk_id）
- 支持相似度检索

#### 阶段4：创建检索模块（src/retriever.py）
- 基于用户问题进行向量检索
- 支持多种检索模式（实体、关系、混合、上下文增强）
- 结果排序和过滤

#### 阶段5：创建问答引擎（src/qa_engine.py）
- 结合检索到的知识生成回答
- 支持多种 LLM（Ollama、OpenAI）
- 支持对话历史
- 支持引用来源

#### 阶段6：创建命令行工具（qa_cli.py）
- 构建向量索引：`python qa_cli.py --build-index`
- 交互式问答：`python qa_cli.py --interactive`
- 单次问答：`python qa_cli.py --query "Who is Alex Mercer?"`

#### 阶段7：更新依赖文件（requirements.txt）
- 添加 sentence-transformers>=2.2.0
- 添加 chromadb>=0.4.0

### 数据流程
```
原始数据文件（CSV）→ 文本生成模块 → JSON 文件（带文本描述）
                                              ↓
                                      嵌入模块 → 向量数据库（ChromaDB）
                                              ↓
                                      检索模块 → 问答引擎 → 最终答案
```
