### 具体实现细节

#### 1. config.py 修改
```python
# 在ProcessingConfig类中添加
enable_llm_logging: bool = Field(default=False)

# 在Config.from_env()中添加读取逻辑
processing=ProcessingConfig(
    max_retries=int(os.getenv("MAX_RETRIES", "3")),
    retry_delay=int(os.getenv("RETRY_DELAY", "2")),
    batch_process_size=int(os.getenv("BATCH_PROCESS_SIZE", "10")),
    enable_entity_linking=os.getenv("ENABLE_ENTITY_LINKING", "false").lower() == "true",
    entity_types_to_link=[t.strip() for t in os.getenv("ENTITY_TYPES_TO_LINK", "").split(",") if t.strip()] if os.getenv("ENTITY_TYPES_TO_LINK", "").strip() else None,
    enable_llm_logging=os.getenv("ENABLE_LLM_LOGGING", "false").lower() == "true"
)
```

#### 2. main.py 配置合并
```python
# 在配置合并部分添加
enable_llm_logging = args.enable_llm_logging or config.processing.enable_llm_logging
```

#### 3. print_config 函数更新
```python
# 添加新的配置项显示
logging.info(f"Enable Entity Linking: {config.processing.enable_entity_linking}")
if config.processing.entity_types_to_link:
    logging.info(f"Entity Types to Link: {', '.join(config.processing.entity_types_to_link)}")
logging.info(f"Enable LLM Logging: {config.processing.enable_llm_logging}")
```

#### 4. .env.example 更新
```env
# Entity Linking Configuration
ENABLE_ENTITY_LINKING=true
ENTITY_TYPES_TO_LINK=Patient,PatientId,Diagnosis

# LLM Logging Configuration
ENABLE_LLM_LOGGING=false
```

#### 5. README.md 新增章节
- **高级配置**：说明环境变量和命令行参数的配置方式
- **实体链接使用场景**：
  - 场景1：增量构建患者病历（避免重复创建患者）
  - 场景2：多类型实体链接（链接患者、诊断、症状等）
  - 场景3：仅链接患者ID（最小化数据库查询）
- **LLM详细日志记录**：说明日志记录的作用、文件位置和使用场景
