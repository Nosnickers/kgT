# 模型未安装问题解决方案

## 问题分析
终端显示错误：`model "mistral:7b-instruct-v0.3-q4_0" not found, try pulling it first (status code: 404)`

- 当前 .env 配置：`OLLAMA_MODEL=mistral:7b-instruct-v0.3-q4_0`
- 已安装模型：`deepseek-r1:14b` 和 `deepseek-r1:8b`

## 解决方案选择

### 方案 1：拉取配置的 Mistral 模型（推荐）
**优点**：使用设计时推荐的模型，效果更稳定
**缺点**：需要下载约 4GB 模型文件

### 方案 2：使用已有的 DeepSeek 模型
**优点**：无需额外下载，节省时间和磁盘空间
**缺点**：模型可能与设计时的预期有所不同

## 实施步骤

### 方案 1：拉取 Mistral 模型
1. 运行拉取命令：
   ```bash
   ollama pull mistral:7b-instruct-v0.3-q4_0
   ```
2. 等待模型下载完成
3. 验证模型已安装：
   ```bash
   ollama list
   ```
4. 继续使用当前配置构建知识图谱

### 方案 2：使用 DeepSeek 模型
1. 编辑 .env 文件：
   - 将 `OLLAMA_MODEL=mistral:7b-instruct-v0.3-q4_0`
   - 修改为 `OLLAMA_MODEL=deepseek-r1:8b`（或 `deepseek-r1:14b`）
2. 保存文件
3. 继续使用新配置构建知识图谱

## 验证和使用
无论选择哪种方案，验证模型安装后，都可以继续构建知识图谱：
```bash
python main.py --build
```

## 建议
- 建议使用方案 1（拉取 Mistral 模型），因为这是设计时推荐的模型，效果更稳定
- 如果磁盘空间有限或下载速度慢，可以选择方案 2
- 模型下载完成后，可以使用 `ollama list` 验证安装状态
- 可以使用 `ollama show <model_name>` 查看模型详情
