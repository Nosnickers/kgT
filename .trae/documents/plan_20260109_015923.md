## 问题分析
从日志中可以看到，实体名称和类型中包含多余的引号，导致在写入Neo4j数据库时出现约束冲突：
```
Node(7) already exists with label `Entity` and property `name` = ''Apple 2030''
```

这表明实体名称中包含了多余的引号，例如 `''Apple 2030''` 而不是正确的 `Apple 2030`。

## 解决方案
在实体提取和处理的各个阶段添加清理逻辑，去除字符串两端的单引号和双引号。

### 实现步骤

1. **修改 `entity_extractor.py` 文件**：
   - 在 `extract` 方法中，添加清理逻辑，去除实体和关系名称、类型两端的引号
   - 确保解析后的实体和关系数据不包含多余的引号

2. **修改 `graph_builder.py` 文件**：
   - 在实体处理阶段，添加额外的清理逻辑
   - 在创建 `Entity` 和 `Relationship` 对象之前，再次清理名称和类型
   - 确保用于去重的实体键也经过清理

3. **修改 `neo4j_manager.py` 文件**（可选）：
   - 在写入数据库之前，添加最后的清理保障
   - 确保最终写入数据库的数据不包含多余引号

### 清理逻辑
使用 Python 的 `strip()` 方法去除字符串两端的引号，包括：
- 单引号 `'`
- 双引号 `"`
- 混合引号 `'"` 或 `"'`

### 关键代码修改

1. **在 `entity_extractor.py` 中**：
   ```python
   # 清理实体和关系数据中的多余引号
   def clean_entity_data(self, entity_data):
       entity_data['name'] = entity_data['name'].strip('"\'')
       entity_data['type'] = entity_data['type'].strip('"\'')
       if 'description' in entity_data:
           entity_data['description'] = entity_data['description'].strip('"\'')
       return entity_data
   ```

2. **在 `graph_builder.py` 中**：
   ```python
   # 清理实体名称和类型
   entity_name = entity.name.strip('"\'')
   entity_type = entity.type.strip('"\'')
   ```

## 预期结果
- 实体名称和类型将不再包含多余的引号
- 写入数据库时不会出现约束冲突
- 知识图谱的实体和关系数据将更加准确和一致
